# Checkpoint: Render script progress freeze (2026-06-06)

## What changed

- Updated `scripts/render_raylib.py` to allow separate Y padding for the pressure field domain:
  - `_field_domain(points, x_margin_factor, y_margin_factor)` now supports independent Y expansion.
  - `_resolve_field_domain(..., y_margin_factor=...)` now accepts and forwards the optional Y factor.
  - Added CLI argument `--field-margin-y`:
    - `None` (default) keeps legacy behavior (X and Y margins match).
    - Set `--field-margin-y` explicitly to expand only the field-domain Y extent while preserving X span.
  - `_evaluate_sweep` now passes `args.field_margin_y` into domain resolve.

## Working render command

Use this to expand the pressure field vertically while keeping airfoil thickness/sizing stable:

```bash
cd /home/claude/xfoil
UV_CACHE_DIR=/tmp/uv-cache DISPLAY=:0 PYTHONPATH=src \
uv run python scripts/render_raylib.py \
  --naca 2412 \
  --alpha-start 4 --alpha-stop 4 --alpha-step 1 \
  --field-origin-mode leading_edge \
  --field-rect-mode geometry \
  --field-domain-mode padded \
  --field-margin 1.8 \
  --field-margin-y 400 \
  --field-mode pressure \
  --field-strength 4.5 \
  --field-core 0.08 \
  --strict-display
```

For static output:

```bash
UV_CACHE_DIR=/tmp/uv-cache PYTHONPATH=src uv run python scripts/render_raylib.py \
  --offscreen \
  --offscreen-output /tmp/test_pressure_field.png \
  <same flags as above>
```
